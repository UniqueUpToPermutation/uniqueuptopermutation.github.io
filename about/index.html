The Purely Academic

My thesis work revolves around the following question: Suppose that we have a matrix \( \mathbf{A} \) that is corrupted by noise. This may be for a number of reasons - but principally I'm interested in linear systems that must be estimated from incomplete or noisy data. This is relatively common across computational physics and machine learning. In practice, this means that instead of having access to \( \mathbf{A} \) itself, we have access to an estimator \( \mathbf{\hat{A}} \). And therefore, naively, instead of solving the linear system
$$ \mathbf{A} \mathbf{x} = \mathbf{b}\,,$$
one is instead solving the linear system
$$ \mathbf{\hat{A}} \mathbf{\hat{x}} = \mathbf{b}\,.$$
Obviously, this means that there will always be an error between the true solution \( \mathbf{x} \) we want and the solution \( \hat{\mathbf{x}} \) we can compute via a linear system solve from the available data. This raises the following fundamental question of my thesis: is there any transformation we can apply to \( \mathbf{\hat{A}} \) that will make the solution \( \mathbf{\hat{x}} \) more accurate?

The short answer is yes.

I've also done some work in the area of model order reduction. In short, many high-fidelity physics models require a huge amount of computing resources - think simulating the drag force across an airplane. Practically, this means that important tasks like uncertainty quantification and Bayesian inference are computationally intractable, since they require not just one, but many solves of the underlying high-fidelity model. Reduced order modelling techniques use data to build fast approximate models as proxies for the original high fidelity physics model. These proxies are useful for the aforementioned situations where one needs to query the high fidelity physics model many times in succession for roughly similar parameters.

Unfortunately, the field has been plagued by the fact that there are usually no guarantees that the output of a reduced order model is actually correct. Unlike in machine learning proper, where there is usually not a ground truth to compare oneself to outside of the data, in model order reduction, one actually has access to the true underlying model. In data science terms: most of the current techniques cannot guarantee that the reduced order models generalize. For problems where correctness is critical (think building a bridge), this is often a deal-breaker.

Some of my research has been on building online algorithms that will update reduced order models during deployment to efficiently ensure they output a correct solution to a certain tolerance. I also maintain a GitHub with a number of computer graphics related projects. In particular, I'm currently working on a real-time physically-based graphics engine called Morpheus. Here's a few screen shots:

While it is nowhere close to ready for production of actual games, it is intended more as a virtual playground for me. Morpheus features:

  • An entity-component-system data-oriented design.
  • A multithreaded task system for updates, rendering, and loading resources.
  • A resource management system for textures, materials, pipelines, shaders, and geometry.
  • A physically-based renderer with spherical harmonics irradiance.
  • An EDitor (work in progress, probably not as good as the standard editor).

I plan to write a number of tutorials about the implementation of physically-based rendering; since I found most of the available resources to be lacking in clarity and/or completeness. If you want, you can find more about Morpheus under Projects.
